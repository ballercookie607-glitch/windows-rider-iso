# Contributing to Windows Rider ISO

Thank you for your interest in contributing! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/windows-rider-iso.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`

## Development Guidelines

### Code Style
- PowerShell scripts should follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) principles adapted for PowerShell
- Use descriptive variable names
- Add comments for complex logic
- Include error handling

### Testing
Before submitting a PR:
1. Test on Windows 10/11
2. Verify with both Community and Professional editions
3. Check for disk space and dependencies
4. Run scripts with admin privileges

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Reference issues: "Fixes #123"

## Submitting Changes

1. Push to your fork
2. Create a Pull Request with a clear description
3. Include any relevant issue numbers
4. Wait for review and address feedback

## Reporting Issues

- Check existing issues first to avoid duplicates
- Provide detailed error messages and logs
- Include your Windows and PowerShell version
- Specify Rider edition (Professional/Community)

## Questions?

Open an issue with the `question` label or start a discussion.

Thank you for contributing!
